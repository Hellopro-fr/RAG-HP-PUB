## 🎯 Objectif

Délivrer un moteur de recherche produits HelloPro **rapide (< 2 secondes)** et
**pertinent** (top-1 conforme à l'intention utilisateur), pour remplacer

## 🏗️ Architecture mise en place

Au lieu d'un seul moteur qui essaie de tout faire (Milvus seul = lent et bruyant),
on a **3 couches qui collaborent** :

```
   ┌────────────────────────────────────────────────────────────┐
   │  COUCHE 1 — Solr V2 (PAGE 1)                                │
   │  Recherche textuelle ultra-rapide (< 200 ms)                │
   │  Garde-fous qualité : HEALTHY / HYBRID / TRASH              │
   └─────────────────────┬──────────────────────────────────────┘
                         │ Si résultat dégradé → on enrichit
                         ▼
   ┌────────────────────────────────────────────────────────────┐
   │  COUCHE 2 — Typesense Hybride (PAGES 2-4 + extensions P1)   │
   │  Recherche par mots-clés + sens (intelligence sémantique)   │
   │  Cerveau Python qui ré-ordonne intelligemment :             │
   │    • Reconnaît les marques (Ritmo, Delabie, Apreau)         │
   │    • Comprend les synonymes (crane = grue, lockers = casier)│
   │    • Pondère les mots rares (médicale > armoire)            │
   │    • Pénalise les résultats incomplets (1 mot sur 3)        │
   └─────────────────────┬──────────────────────────────────────┘
                         │ Si lent ou indisponible → fallback
                         ▼
   ┌────────────────────────────────────────────────────────────┐
   │  COUCHE 3 — Filet de sécurité (BM25 pur)                    │
   │  Si la couche 2 dépasse 30 sec, on bascule auto             │
   │  → 0% d'erreur visible côté front                           │
   └────────────────────────────────────────────────────────────┘
```

**Le service tourne sur Google Cloud (GKE)**, avec 2 pods en redondance pour
absorber la charge et garantir la disponibilité.

## 🚀 Les 5 grandes optimisations qui changent tout

| # | Nom | Ce que ça résout |
|---|---|---|
| **A3** | Seuil adaptatif par catégorie | Les requêtes génériques (compresseur, ERP) ne sont plus filtrées trop strictement |
| **A4** | Pondération des mots rares (IDF) | "Soudure **Ritmo**" trouve enfin le bon Ritmo, pas un autre |
| **A6** | Synonymes intelligents | "e-crane" trouve les grues, "lockers" trouve les casiers (anglicismes B2B) |
| **A7** | Couverture stricte | "Barre **laser** à **LED**" ne sort plus les scanners code-barre |
| **A8** | Marque comme contrainte | "Urinoir **Delabie**" sort un urinoir Delabie, pas un distributeur Delabie |

Plus : un **filet de sécurité automatique** qui élimine les erreurs visibles
côté utilisateur (PR #639).

## 🛡️ Robustesse opérationnelle

Plusieurs mécanismes assurent que **le moteur reste opérationnel même en cas
de panne d'un composant** :

| Risque | Mitigation en place |
|---|---|
| Service IA (embedding) lent ou down | **Fallback automatique** vers recherche textuelle pure (PR #639) |
| Crash d'un pod GKE | **2 pods redondants** + restart automatique Kubernetes |
| Mise à jour catalogue produits | **Synchronisation quotidienne** automatique (cron) |
| Évolution du vocabulaire | **Recalcul hebdomadaire** des poids des mots (cron dimanche 4h) |
| Régression qualité | **Rollback en 5 minutes** via paramètre URL `?legacy=1` ou flag PHP |
| Catastrophe GKE | **Repointer la gateway** sur ancien backend Milvus en ~10 min |

logicielle + la collaboration entre composants existants.
