Guide d'installation des commandes MCP

Instructions pour chaque runtime et chaque OS

npx - Node.js Package Executor

npx est inclus avec npm (Node.js). Il permet d'exécuter des paquets npm sans installation globale. C'est la commande par défaut et la plus courante pour les clients MCP.

Installation sur Windows

Option 1 : Via winget (recommandé)

winget install OpenJS.NodeJS.LTS

Option 2 : Via Chocolatey

choco install nodejs-lts

Option 3 : Installeur officiel

Téléchargez le .msi depuis nodejs.org

Installation sur Linux

Option 1 : Via NodeSource (recommandé)

curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

Option 2 : Via nvm (Node Version Manager)

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install --lts

Option 3 : Via Snap

sudo snap install node --classic

Installation sur macOS

Option 1 : Via Homebrew (recommandé)

brew install node

Option 2 : Via nvm

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.zshrc
nvm install --lts

Option 3 : Installeur officiel

Téléchargez le .pkg depuis nodejs.org

Vérification

node --version   # v18+ requis
npx --version

Utilisation avec MCP Gateway

Ajoutez cette configuration dans votre fichier .mcp.json :

{
  "mcpServers": {
    "hellopro-gateway": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://<gateway-url>/mcp",
        "--header",
        "X-MCP-Scope-Token: <votre-token>"
      ]
    }
  }
}
Note : Le flag -y accepte automatiquement l'installation du paquet mcp-remote sans confirmation. Ajoutez --allow-http si votre gateway utilise HTTP (non-HTTPS).

bunx - Bun Package Executor

bunx est l'équivalent de npx pour le runtime Bun. Plus rapide que npx, il est compatible avec les paquets npm.

Installation sur Windows

Via PowerShell (recommandé)

irm bun.sh/install.ps1 | iex

Via npm

npm install -g bun

Installation sur Linux

Via script officiel (recommandé)

curl -fsSL https://bun.sh/install | bash

Via npm

npm install -g bun

Installation sur macOS

Via Homebrew (recommandé)

brew install oven-sh/bun/bun

Via script

curl -fsSL https://bun.sh/install | bash

Vérification

bun --version
bunx --version

Utilisation avec MCP Gateway

{
  "mcpServers": {
    "hellopro-gateway": {
      "command": "bunx",
      "args": [
        "mcp-remote",
        "https://<gateway-url>/mcp",
        "--header",
        "X-MCP-Scope-Token: <votre-token>"
      ]
    }
  }
}
Avantage : bunx ne nécessite pas le flag -y — il installe automatiquement les paquets manquants.

deno - Deno Runtime

Deno est un runtime JavaScript/TypeScript sécurisé par défaut, avec support natif des paquets npm via le préfixe npm:.

Installation sur Windows

Via PowerShell (recommandé)

irm https://deno.land/install.ps1 | iex

Via winget

winget install DenoLand.Deno

Via Chocolatey

choco install deno

Installation sur Linux

Via script officiel (recommandé)

curl -fsSL https://deno.land/install.sh | sh

Via Snap

sudo snap install deno

Installation sur macOS

Via Homebrew (recommandé)

brew install deno

Via script

curl -fsSL https://deno.land/install.sh | sh

Vérification

deno --version

Utilisation avec MCP Gateway

{
  "mcpServers": {
    "hellopro-gateway": {
      "command": "deno",
      "args": [
        "run",
        "--allow-all",
        "npm:mcp-remote",
        "https://<gateway-url>/mcp",
        "--header",
        "X-MCP-Scope-Token: <votre-token>"
      ]
    }
  }
}
Note : Le flag --allow-all accorde toutes les permissions au script. Pour plus de sécurité, utilisez --allow-net --allow-read --allow-env.

uvx - Python Package Executor (uv)

uvx est l'exécuteur de paquets de uv, le gestionnaire Python ultra-rapide d'Astral. Il utilise le paquet mcp-proxy pour se connecter au gateway.

Installation sur Windows

Via PowerShell (recommandé)

irm https://astral.sh/uv/install.ps1 | iex

Via winget

winget install astral-sh.uv

Via pip

pip install uv

Installation sur Linux

Via script officiel (recommandé)

curl -LsSf https://astral.sh/uv/install.sh | sh

Via pip

pip install uv

Installation sur macOS

Via Homebrew (recommandé)

brew install uv

Via script

curl -LsSf https://astral.sh/uv/install.sh | sh

Vérification

uv --version
uvx --version

Utilisation avec MCP Gateway

{
  "mcpServers": {
    "hellopro-gateway": {
      "command": "uvx",
      "args": [
        "mcp-proxy",
        "https://<gateway-url>/mcp",
        "--header",
        "X-MCP-Scope-Token: <votre-token>"
      ]
    }
  }
}
Note : uvx utilise le paquet Python mcp-proxy (et non mcp-remote qui est un paquet npm). L'API est compatible.

docker - Conteneur isolé

Exécute le client MCP dans un conteneur Docker isolé. Aucune dépendance locale (Node.js, Python, etc.) n'est requise — seul Docker doit être installé.

Installation sur Windows

Docker Desktop (recommandé)

Téléchargez depuis docker.com

Prérequis : WSL 2 activé

Via winget

winget install Docker.DockerDesktop

Installation sur Linux

Via script officiel (recommandé)

curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
newgrp docker

Via apt (Ubuntu/Debian)

sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl enable --now docker

Installation sur macOS

Docker Desktop (recommandé)

Téléchargez depuis docker.com

Via Homebrew

brew install --cask docker

Vérification

docker --version
docker run hello-world

Utilisation avec MCP Gateway

{
  "mcpServers": {
    "hellopro-gateway": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "ghcr.io/anthropics/mcp-remote",
        "https://<gateway-url>/mcp",
        "--header",
        "X-MCP-Scope-Token: <votre-token>"
      ]
    }
  }
}
Flags importants :
  • -i : mode interactif (requis pour stdin/stdout MCP)
  • --rm : supprime le conteneur après l'arrêt
