# QC-tracking-service

Web UI for browsing and searching QC pipeline tracking files generated by other QC services.

## Tech Stack

- Python 3.10
- FastAPI + uvicorn
- Static HTML frontend (served by FastAPI)

## Build / Run

- **Docker-only build** (context = repo root):
  ```
  docker build -f apps-microservices/QC-tracking-service/Dockerfile .
  ```
- Entrypoint: `uvicorn main:app --host 0.0.0.0 --port 8590`
- Port: **8590**
- Tracking files mounted via Docker volume at `TRACKING_BASE_PATH` (default `/app/tracking`)

## Folder Structure

```
QC-tracking-service/
  main.py              # FastAPI app with all endpoints
  Dockerfile
  requirements.txt
  static/
    index.html         # SPA frontend
  app/
    __init__.py
```

## API Endpoints

| Method | Path            | Description                                    |
|--------|-----------------|------------------------------------------------|
| GET    | /               | Serves the main HTML page                      |
| GET    | /api/browse     | Browse directory contents (paginated, limit=20)|
| GET    | /api/file       | Read file content (UTF-8)                      |
| GET    | /api/download   | Download a file                                |
| GET    | /api/search     | Search files by name or content (min 2 chars)  |

## Dependencies on Other Services

- **QC pipeline services**: reads their tracking files via shared Docker volume
- No RabbitMQ dependency
- No LLM dependency

## Conventions

- Path sanitization prevents directory traversal attacks
- Files sorted by modification time (most recent first)
- Directories always shown; file listing paginated
